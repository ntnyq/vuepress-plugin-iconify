# AGENTS.md

Guidance for AI coding agents working in this repository.

## Project Snapshot

- Package: `vuepress-plugin-iconify`, a VuePress 2 plugin for Iconify icons.
- Workspace layout: root TypeScript library package plus the `docs/` VuePress demo/docs package.
- Package manager: `pnpm` (`packageManager: pnpm@11.8.0`).
- Public usage, options, and examples live in [README.md](README.md) and [docs/guide/README.md](docs/guide/README.md); link to those instead of duplicating long usage text.

## Commands

Run from the repository root unless noted.

- Install dependencies: `pnpm install`
- Build library: `pnpm run build`
- Watch library bundle: `pnpm run dev`
- Type check: `pnpm run typecheck`
- Lint: `pnpm run lint`
- Format: `pnpm run format`
- Check formatting: `pnpm run format:check`
- Test: `pnpm run test`
- Docs dev server: `pnpm run docs:dev`
- Docs build: `pnpm run docs:build`
- Release validation: `pnpm run release:check`

## Source Boundaries

- `src/node/`: VuePress plugin entry and build-time injection. `src/node/iconifyPlugin.ts` defines the plugin, loads static icons, injects `__ICONIFY_*` constants, and points VuePress to the client config.
- `src/client/`: Vue runtime integration. `src/client/config.ts` registers static icons with Iconify and installs the global component; `src/client/components/Icon.ts` wraps `@iconify/vue`.
- `src/shared/`: shared options, constants, and file/icon collection utilities used by both node and client code.
- `tests/`: Vitest coverage for shared utility behavior such as icon id extraction and scan file collection.

Keep node/plugin concerns out of client files, and keep runtime Vue concerns out of `src/node/**`.

## Build And Exports

- Bundling is configured in [tsdown.config.ts](tsdown.config.ts) with `dts: true`, `unbundle: true`, and entries for `src/node/**`, `src/client/**`, and `src/shared/**`.
- Public package exports are declared in [package.json](package.json). When adding, moving, or removing public entry points, update both the tsdown entries and package exports together.
- The built `dist/` output is generated by tsdown and should not be treated as source.

## Icon Loading Pitfalls

- Offline mode is the default. Missing collected icons render nothing client-side and are warned about during static icon loading.
- Static icons are resolved at build time from installed `@iconify-json/<prefix>` packages via a `require` rooted at the consuming project.
- Auto-scan looks for `icon="prefix:name"`, `icon='prefix:name'`, and `~prefix:name~` in configured scan files. Update tests if changing regexes in `src/shared/constants.ts`.
- `staticIcons.collections[prefix] = true` expands a whole collection during resolution; selected names and explicit `icons` are collected separately first.
- Keep support for both string icon ids and raw `IconifyIcon` objects in the client component.

## Style And Tooling

- Use TypeScript ESM with explicit `.js` import specifiers for local source imports that compile to ESM output.
- Formatting is `oxfmt`, not Prettier. ESLint is configured via `@ntnyq/eslint-config` in [eslint.config.mjs](eslint.config.mjs).
- Maintain strict TypeScript compatibility from [tsconfig.json](tsconfig.json), including `moduleResolution: "Bundler"`, `isolatedModules`, and `verbatimModuleSyntax`.
- Add or update focused Vitest tests for shared parsing, scanning, option, or icon resolution behavior.

## Docs Updates

- If plugin options, defaults, public component behavior, or offline/online mode semantics change, update [README.md](README.md) and [docs/guide/README.md](docs/guide/README.md).
- The docs package depends on `vuepress-plugin-iconify` via `workspace:*`, so `pnpm run docs:dev` exercises the local package.

## Before Finishing

Run the narrowest useful validation for the change. For library behavior changes, prefer:

1. `pnpm run lint`
2. `pnpm run typecheck`
3. `pnpm run test`
4. `pnpm run build`

For docs-facing behavior, also run `pnpm run docs:build` when practical.
